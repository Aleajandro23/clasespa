Resumen del Proyecto

____________________________________________________

El proyecto comenzó con la recolección de datos de múltiples fuentes, abarcando el período de 2016 a 2024. Utilizamos csvkit para convertir hojas de Excel a formato CSV, facilitando su manejo. Para los datos provenientes de diferentes años, desarrollamos scripts de partición específicos, asegurando una organización eficiente.

Normalización de Esquemas:

Un desafío significativo fue la variación en los formatos de datos a lo largo de los años. Para abordar esto, creamos esquemas personalizados que garantizaron la consistencia en las columnas de datos. Este proceso fue crucial para unificar la información y hacerla comparable a lo largo del tiempo.

Procesamiento Avanzado de Datos:

Empleamos una combinación de tecnologías para el procesamiento de datos: Spark y Scala: Utilizados para el manejo inicial de grandes volúmenes de datos. Python con Pandas: Implementado para datos más recientes, permitiendo una limpieza y transformación más detallada. Esta combinación nos permitió adaptar nuestro enfoque a las características específicas de cada conjunto de datos.

Optimización del Almacenamiento:

Adoptamos el formato Parquet para el almacenamiento de datos unificados. Esta decisión mejoró significativamente la eficiencia en el almacenamiento y aceleró el acceso a los datos, crucial para análisis posteriores.

Diseño de la Base de Datos:

El diseño de la base de datos se realizó en dos etapas clave: - Modelo Conceptual: Desarrollamos un Modelo Entidad-Relación (E-R) detallado. - Modelo Lógico: Transformamos el modelo E-R en un Modelo Relacional, optimizado para su implementación en MySQL.

Implementación de la Base de Datos:

Generamos un script SQL completo para crear la estructura de la base de datos en MySQL. Este script incluyó la definición de todas las tablas, relaciones y restricciones necesarias para mantener la integridad de los datos.

Carga de Datos:

La carga de datos se realizó en dos fases: - Datos Base: Importamos la información principal sobre detenciones y detenidos. - Datos Complementarios: Incorporamos variables socioeconómicas adicionales para enriquecer el análisis.

Preparación para Análisis:

Como paso final, generamos un archivo CSV consolidado que incluye todas las variables relevantes. Este archivo está diseñado para facilitar análisis futuros y la integración con otras herramientas de análisis de datos.

public static void ingresarValoresMatriz(int[][] matriz, String nombre, Scanner scanner): Este método se utiliza para ingresar los valores de los elementos de una matriz. Recibe como argumentos la matriz a la que se ingresarán los valores, un nombre para mostrar en los mensajes de entrada y un objeto Scanner para leer la entrada del usuario.

public static int[][] multiplicarMatrices(int[][] matrizA, int[][] matrizB): Este método realiza la multiplicación de dos matrices y devuelve la matriz resultado. Toma dos matrices como argumentos y utiliza bucles for anidados para realizar la multiplicación de los elementos.

public static void imprimirMatriz(int[][] matriz): Este método se utiliza para imprimir una matriz en la consola. Recibe una matriz como argumento y utiliza bucles for para recorrer y mostrar los elementos de la matriz.

El código completo realiza los siguientes pasos: Solicita al usuario ingresar el tamaño de las matrices. Verifica si las matrices se pueden multiplicar (el número de columnas de la matriz A debe ser igual al número de filas de la matriz B). Si no se cumple esta condición, muestra un mensaje de error y finaliza el programa. Crea las matrices A y B con los tamaños especificados. Solicita al usuario ingresar los valores de los elementos de las matrices A y B. Llama al método multiplicarMatrices para realizar la multiplicación y obtiene la matriz resultado. Imprime las matrices A, B y el resultado de la multiplicación utilizando el método imprimirMatriz. Este código es un ejemplo básico de multiplicación de matrices en Java, que puede servir como punto de partida para desarrollar programas más complejos relacionados con el álgebra lineal.

Desafíos y Soluciones:

_________________________________________________________________________________________________

Inconsistencia de Datos

Superamos las variaciones en los formatos de datos a lo largo de los años mediante un proceso riguroso de normalización.

Volumen de Datos

Utilizamos tecnologías como Spark y Parquet para manejar eficientemente grandes cantidades de información.

Integración de Fuentes Diversas

Desarrollamos métodos personalizados para unificar datos de múltiples fuentes y años en un esquema coherente.

_________________________________________________________________________________
RESULTADO FINAL
_________________________________________________________________________________

El proyecto culminó con una base de datos MySQL robusta y completamente funcional, que integra datos de detenciones de 2016 a 2024 junto con variables socioeconómicas relevantes. Esta base de datos está preparada para soportar una amplia gama de análisis, desde tendencias temporales hasta correlaciones complejas entre factores socioeconómicos y patrones de detención.

Esta infraestructura de datos proporciona una base sólida para investigaciones futuras en seguridad pública, criminología y políticas de justicia en Ecuador, ofreciendo una visión integral y longitudinal de los patrones de detención en el país.
